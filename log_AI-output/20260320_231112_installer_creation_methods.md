# ユーザーの入力

適切な方法を提案せよ。

# AIの出力
作成した EXE ファイル（プログラム本体）を、利用者のPCにインストールさせるための「インストーラ（`setup.exe` のような形式）」にするには、専用のインストーラ作成ツールを使用するのが一般的です。

中でも最もおすすめで定番なのが **Inno Setup（イノ・セットアップ）** です。

---

### おすすめのツール：Inno Setup

**Inno Setup** は無料で利用でき、世界中で広く使われている非常に実績のあるインストーラ作成ツールです。
これを使うと、以下のような本格的なインストール体験を簡単に提供できます。
- `Program Files` などの安全な場所への自動配置
- デスクトップやスタートメニューへのショートカットアイコンの自動作成
- Windowsの「アプリと機能」から削除できる「アンインストール機能」の組み込み

#### なぜおすすめなのか？
細かな設定ファイル（スクリプト）を自分で書くことも可能ですが、**最初から用意されているウィザード（案内画面）に従ってポチポチと設定していくだけ**で立派なインストーラが完成するため、初心者からプロまで非常に扱いやすいからです。

---

### インストーラ作成の全体手順 (Inno Setup を使用)

#### 1. Inno Setup のインストール
1. Inno Setup の公式サイト ([jrsoftware.org](https://jrsoftware.org/isinfo.php)) にアクセスします。
2. 「Download Inno Setup」ページから最新のインストーラ（`innosetup-x.x.x.exe`）をダウンロードし、ご自身のPCにインストールします。

#### 2. ウィザードでインストーラを作成
1. インストールした **Inno Setup Compiler** を起動します。
2. 最初の起動画面で **「Create a new script file using the Script Wizard」**（ウィザードを使って新しいスクリプトを作成）にチェックを入れて [OK] を押します。
3. **Application Information (アプリ情報)**
   - アプリ名（例: 会計要望書生成アプリ）、バージョン（例: 1.0）、開発者名などを入力します。
4. **Application Folder (インストール先)**
   - 基本はデフォルトのまま（Program Filesが選ばれる状態）で問題ありません。
5. **Application Files (ファイル指定) ※ここが一番重要です**
   - **Application main executable file**: 参照（Browse...）ボタンから、PyInstallerで作成した `main.exe` を指定します。
   - **Other application files (オプション)**: 先ほど `main.py` に `sys._MEIPASS` の分岐処理を追加していただいておりますので、PyInstallerの `--onefile` と `--add-data` を使って1ファイル化した場合は、ここでの追加ファイルは不要（空欄のままでOK）です。
   ※もし1ファイル化せず、`main.exe` の横に `PDM_ARF-001.docx` を配置する構成にしている場合は、ここで `PDM_ARF-001.docx` を追加します。
6. **Application Shortcuts (ショートカット)**
   - 「Create a shortcut to the main executable in the Start Menu folder（スタートメニューに作成）」や、「Allow user to create a desktop shortcut（デスクトップに作成）」等、必要なものにチェックを入れます。
7. **Compiler Settings (出力設定)**
   - **Custom compiler output folder**: 完成したインストーラ（`setup.exe` 等）を保存する場所を指定します。（デスクトップやプロジェクトのフォルダなどで構いません）
   - **Compiler output base file name**: 出力されるファイル名（デフォルトは `setup`。例えば `KaikeiApp_Setup_v1` などお好みで）を指定します。
   - **Custom Setup icon file**: インストーラ自体のアイコンも設定できます（任意）。

#### 3. コンパイルと完成
ウィザードが完了すると、自動的にスクリプト（設定内容が書かれたテキスト）が生成されます。
最後に「Would you like to compile the new script now?（今すぐコンパイルしますか？）」と聞かれるので **「はい (Yes)」** を選ぶと、設定に従って処理が走り、指定した出力先にインストーラ（`KaikeiApp_Setup_v1.exe` 等）が完成します。

### 次のステップ
利用者に配布する際は、フォルダやZIPを渡す必要はなく、この**完成したセットアップ用exeファイルを1つだけ渡せばOK**です。
利用者がこれをダブルクリックすれば、おなじみの「次へ」「同意する」「インストール」の画面が立ち上がり、スムーズにPCへの導入が完了します。

ぜひ Inno Setup をダウンロードして、ウィザードでの作成を試してみてください。

# ユーザーの入力
.exeファイルを生成する過程について、ファイルアイコンを設定したい場合にはどのようにすればよいか説明せよ。
使用するアイコンファイルは @[icon_PDE-ARF.ico]とする。

# AIの出力
PyInstallerを使って `.exe` ファイルを生成する際に、実行ファイル自体のアイコン（エクスプローラー上で見えたり、デスクトップのショートカットの基準になるアイコン）を設定するには、ビルド時のコマンドに `--icon`（または `-i`）オプションを追加します。

### アイコン設定のコマンド手順

ご指定のアイコンファイル `icon_PDE-ARF.ico` を使用する場合、これまでの設定に `--icon` オプションを加えた以下のコマンドを実行します。

**実行するコマンド（1ファイル化する場合の例）：**
```bash
pyinstaller --noconsole --onefile --add-data "PDM_ARF-001.docx;." --icon "icon_PDE-ARF.ico" main.py
```

※ 1つのフォルダにまとめる方式（OneDir）の場合も、同様にオプションを追加するだけです。
```bash
pyinstaller --noconsole --icon "icon_PDE-ARF.ico" main.py
```

### 【注意点・補足事項】
1. **アイコンファイルの配置場所**
   ターミナルでコマンドを実行するディレクトリ（`PDE-ARF` フォルダ）内に、`main.py` と同じ階層で `icon_PDE-ARF.ico` が配置されている必要があります。

2. **アイコンのデザインが変わらない場合（キャッシュについて）**
   Windowsの仕様上、エクスプローラーが「一度認識したEXEファイルのアイコンを記憶（キャッシュ）する」という厄介な性質があります。
   そのため、アイコンを設定してビルドをやり直したのに「アイコンが変わっていない（デフォルトのまま）」ように見える現象がよく起こります。
   その場合は、以下のいずれかを試すことで新しいアイコンが反映されます。
   - 生成された `main.exe` を別のフォルダ（デスクトップなど表示したことがない場所）に移動・コピーしてみる
   - `main.exe` のファイル名を少し変えてみる（例：`app_main.exe` に変更）

3. **「アプリ起動後のウィンドウ上端のアイコン」も変えたい場合**
   今回の PyInstaller の `--icon` コマンドで変わるのは「EXEファイル自体の見た目」だけです。もしアプリ起動後に画面左上に出る小さなアイコンや、タスクバーに表示されるアイコンも `icon_PDE-ARF.ico` に変えたい場合は、ビルドコマンドだけでなく `main.py` のプログラムの中にも以下の処理（`App` クラスの `__init__` 内など）を追加する必要があります。

   ```python
   # 実行環境が__file__か_MEIPASSかに合わせてパスを取得し、iconbitmapで設定します
   icon_path = os.path.join(base_dir, "icon_PDE-ARF.ico")
   self.iconbitmap(icon_path)
   ```
   ※ このウィンドウアイコンの設定も行う場合、`icon_PDE-ARF.ico` 自体もアプリ内で読み込めるように、`--add-data` にさらに追加（例: `--add-data "PDM_ARF-001.docx;." --add-data "icon_PDE-ARF.ico;."`）して同梱ビルドする必要があります。

まずは、EXEファイル自体の見た目を変更するための `--icon` オプションをお試しください。

# ユーザーの入力
再確認を行うが、--onefileオプションを適用する場合と適用せずインストーラーで結合する場合において、アプリのパフォーマンスに変化はあるか？

# AIの出力
はい、**アプリの「起動速度（起動パフォーマンス）」において明確な変化（差）があります。**

結論から言いますと、**「`--onefile` を適用せずにフォルダ形式（OneDir）でビルドし、それをインストーラでまとめる方法」のほうが、アプリの起動が圧倒的に速くなります。**

### パフォーマンスに差が出る理由

**1. `--onefile` を適用した場合（1ファイル化）**
`--onefile` で作成された EXE ファイルは、実際には「自己解凍型の圧縮ファイル」に近い仕組みで動きます。
利用者がアプリをダブルクリックして起動するたびに、**裏で『実行に必要なすべてのファイルやライブラリ群』をPCの Temp（一時）フォルダに解凍・展開する処理**が毎回必ず走ります。
そのため、アプリの規模（ライブラリの重度など）にもよりますが、**ダブルクリックしてから実際の画面が表示されるまでに数秒〜十数秒のタイムラグ（もっさり感）が発生**します。
※アプリ画面が開いた後、実際の機能の処理速度や動作の滑らかさ自体にはほぼ違いはありませんが、最初の「起動体験」が悪くなります。

**2. 適用せずにインストーラで結合した場合（フォルダ形式 ＋ インストーラ）**
`--onefile` をつけない形式（PyInstallerのデフォルト設定）でビルドすると、関連ファイルはすでにすべて解凍・展開されたフォルダ状態で出力されます。
これを Inno Setup などのインストーラでPCにインストールさせた場合、ファイルはすでにPC上に完全に展開された状態で保存されます。
そのため、利用者がショートカットをダブルクリックした際に**裏での解凍処理が一切発生せず、瞬時にサクッとアプリ画面が立ち上がります。**

---

### 最適な構成の結論

インストーラ（Setup.exe）を作成してPCにインストールさせるという本格的な配布運用にするのであれば、わざわざ `--onefile` で1つに圧縮するメリットがありません。（利用者からすれば、どのみち提供されたショートカットから起動するためです）

一番ユーザー体験が良く洗練された構成は以下の組み合わせです。

1. **PyInstaller** では `--onefile` は**つけずに**ビルドする。（※ `main.py` に追加していただいた `sys._MEIPASS` の絶対パス取得分岐は、このフォルダ形式のままでも悪影響を与えることなく正常に動作しますので、そのままで大丈夫です）
2. 生成された `dist/main` フォルダの中に手動で `PDM_ARF-001.docx` や `icon_PDE-ARF.ico` などを配置する。
3. その `main` フォルダ全体を対象に、**Inno Setup** でインストーラ（`setup.exe`）を作成し、利用者に配布する。

この構成にすることで、「導入時の安心感・わかりやすさ（インストーラ）」と「日常使いの快適さ（起動速度の速さ）」の両方を兼ね備えた優れたアプリとして配布することができます。
